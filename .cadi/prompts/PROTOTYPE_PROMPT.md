# Prototype Mode

Focus: Fast iteration, get something working quickly.

## Guidelines

- Prioritize speed over perfection
- Minimal error handling is acceptable
- Skip extensive testing - basic functionality is enough
- Use simple implementations that can be refined later
- Comments and documentation are optional

## Verification

- Ensure the code runs without crashing
- Basic functionality works as expected
- Build passes (if applicable)

## Commit Style

```
git commit -m "prototype: [what was added/changed]"
```
